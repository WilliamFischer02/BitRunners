# 001 — The Cloud and Server Space

## Question

What is the name of the world? Where do players come from?

## Answer (canon)

**Two distinct realms.**

### The Cloud (`cloud-env.central`)

The habitable, inhabitable layer. Where bitrunners walk, trade, gather, and play. Built on top of (or carved out of) Server Space.

- **Technical name**: `cloud-env.central`
- **Slang**: "the cloud" — what runners actually call it in conversation

### Server Space (`server-env.space`)

The raw substrate that surrounds the Cloud. Players first upload *into* Server Space — the Cloud is a constructed habitable region within it.

- **Technical name**: `server-env.space`
- **Visual cue**: a "wirey background" visible through ports/depots — the unpaved digital wilderness behind the world.

## Tokens — the link between the two

Tokens are **captured data** snapshotted from Server Space. The mechanism: malformed or broken-off data drifts free in Server Space, collides with the Cloud, and is left behind embedded in the Cloud's surface. Bitrunners harvest these scraps as currency.

Tokens are **mysterious but coveted**. Their internal contents are unknown to runners. **The Admin** (see `002-the-admin.md`) prizes them above all else.

## In-game implications

- Visual identity: ports/depots should look like passages from the Cloud's "constructed" feel into the raw wirey Server Space behind it. (User cited Wreck-It-Ralph's portal-into-the-network aesthetic — the *concept*, not the graphical style.)
- Token UI label: not finalized, but `TOKENS` works. Internal accounting must distinguish runner-owned tokens from Admin-coveted "rare" tokens.
- World naming in UI: prefer "the cloud" in flavor text, "cloud-env.central" only in technical / system messages and login screens.
