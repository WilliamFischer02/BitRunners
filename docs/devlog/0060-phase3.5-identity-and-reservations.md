### Sub-Phase B — username overhead label + protocol bump

- **`PROTOCOL_VERSION → 2`** in `@bitrunners/shared`. Single deploy window: server + client must ship together. Old clients soft-warn (no hard handshake gate today).
- `apps/server/src/state.ts` — `PlayerState` gains `displayName`, `equippedBadge`, `equippedTheme`.
- `apps/server/src/sphere-room.ts` — new `'identity'` message handler. Server shape-validates against `isValidDisplayName` / `isValidBadgeKey` / `isValidThemeKey` (added to `@bitrunners/shared`). Ownership of equipped_* is enforced by the SECURITY DEFINER RPCs the client must call BEFORE sending identity — the room trusts what an authenticated client sends here. `onJoin` accepts the same identity fields so first appearance includes them.
- `apps/web/src/network.ts` — `joinSphere` signature now takes `JoinOptions` (className + optional identity fields). New `sendIdentity()` on the session. `RemotePlayer` gains `displayName`, `equippedBadge`, `equippedTheme`. New `onIdentity` callback.
- `apps/web/src/profile.ts` — single source of truth for local identity. Fetches via `get_my_identity` RPC on auth changes. Emits `'bitrunners:identity-changed'`. Guest fallback: deterministic `runner_<6 chars>`.
- `apps/web/src/badges.ts` — 20-entry badge catalog (10 tiers × 2 factions). Glyphs + tints + per-tier Samaritan threshold helper.
- `apps/web/src/scene.ts` —
  - Local floating tag now reads from `profile.ts`. The random session-code is gone. Tag becomes a `<button>` that dispatches `'bitrunners:edit-identity'` on tap.
  - Badge glyph rendered beside the name (tinted per faction).
  - `'!'` micro-dot when `earned_badges.acknowledged = false` for any owned badge.
  - Each remote avatar gets a `.player-tag--remote` DOM node, projected each frame above its head. NPCs get a faint `.player-tag--npc` label with the dweller archetype.
  - On any identity change, `netSession.sendIdentity()` fans out to the room.
- `apps/web/src/UsernameEditor.tsx` — modal opens on the `'bitrunners:edit-identity'` event. Curated 1–2 word composer pulled from `emoticron_dictionary` `category='name'`. Submits via `submit_display_name` RPC. Includes the `BadgeStrip` side-list with two faction ladders, equip / unequip, acknowledged tracking. Guest gate: greyed-out panel + sign-up CTA when not authed.
- `apps/web/src/AdminConsole.tsx` — new `UsernameQueue` section. Lists pending names, owner approves / rejects with optional note. Uses the same admin patterns established in 0006.
- `apps/web/src/supabase.ts` — new client wrappers: `fetchMyIdentity`, `submitDisplayName`, `equipBadge`, `acknowledgeBadge`, `fetchMyBadges`, `fetchDictionary`, `adminListPendingNames`, `adminApproveName`, `adminRejectName`.
- `apps/web/src/style.css` — new `.player-tag--self/.remote/.npc` variants, badge glyph + alert dot styles, badge ladder grid + cells, `prefers-reduced-motion` guard on the alert pulse.
- `apps/web/src/App.tsx` — boots the identity subsystem (`startIdentity()`) on first import, mounts `<UsernameEditor />`.
- `CLAUDE.md` — moderation paragraph amended; cross-links `docs/lore/015-chat-policy.md`.
